Callbacks

The following example will be helpful in understanding how callbacks and how passing functions to other functions works!

Example - Segment 1

Passing functions to other functions

In this example we are going to create a function called myFunc and pass another function called print as an argument to it.

                                        
function myFunc(myCallback) {
    myCallback("Hello World!");
}

function print(value) {
    console.log(value);
}

myFunc(print)
Console: 
                                        
Hello World!

The print function is passed as an argument only when we call myFunc

After executing this code, "Hello World!" you be logged on the console

Let's better understand this code:

  • When we call myFunc we are essentially creating a new variable called myCallback
  • myCallback is itself going to be a new function, and will be assigned with print
  • After this, print is going to receive "Hello World!" as it's value.
  • print is now free to use this value inside of it's scope, and in this case, log it to the console.
Example - Segment 2

Repeating the last segment without using myFunc

Using a variable instead of a function.

                                        
function print(value) {
    console.log(value);
}

const myCallback = print;
myCallback("Hello World!");
Console: 
                                        
Hello World!

This code has the exact same result that myFunc had.

The reason behind this is simple:

  • When calling a function, all we are really doing is mapping an outer variable to the inner variable inside of that function.
  • In our example, we call myFunc and the outer variable print is mapped to the inner variable inside myCallback

The reason that this works is because in JavaScript functions are variables

Do not think of functions as something different from variables!

Functions are not some "new thing" that works differently from everything else!

Functions work in the exact same way as any other variable, and consequently, as any other Object!

Example - Segment 3

Setting values on a function

To testify what has been said, we can show how it is possible to set values to a function just like we can do with Objects.

                                        
function print(value) {
    myName = "John";
}

print.myAge = 32

console.log(print.myAge)

console.dir(print)
Console: 
                                        
John

function print(value)
    arguments: null
    caller: null
    length: 1
    myAge: 32

We can store things inside of functions just like we can do with Objects!

Using console.dir shows us the myAge property as being set on our print function

Example - Segment 4

Most novices think that functions always need to have () after them, but this is not true.

You only need to write () after a function when you want to call that function.

Avoid doing the following mistake

                                        
function print(value) {
    console.log(`This is my value: ${value}`);
}

function myFunc(myCallback) {
    myCallback("Hello World!")
}

myFunc( print() )
Console: 
                                        
This is my value: undefined

Uncaught TypeError: myCallback is not a function

When the compiler tried to run this code, it did not assign the print function to myCallback

Instead, the compiler simply called and executed print

As this was not supposed to happen, the print function did not take the an argument, and therefore value was assigned with undefined

We can see that undefined really was logged in the console

And only after all of this myFunc is going to be executed

As print did not have a return statement, undefined was returned by default

myCallback was then assigned with undefined

The compiler trie to call myCallback using undefined as a callback function and we got an error.

Callbacks must necessarily be functions, what this error is saying is that the value the console got for a callback was not a function

Indeed it wasn't, the value the console got was undefined, and this is not a function

Example - Segment 5

Correcting our mistake

Suppose that, in this example, we want to do 2 things:

  1. Return the Object called myName in it's entirety to the console
  2. Return only the property called age from myName
                                        
function myFunc(person) {
    return console.log(person)
}

const myName = { 
    name: "John",
    age:32
};

//FIRST PART
myFunc(myName)

//SECOND PART
myFunc(myName.age)
Console: 
                                        
Object
    age: 32
    name: "John"
    <prototype>: Object { }

32

For the first part, we simple called myFunc passing myName

For the second part, we passed myName but also specifying the property that we wanted, in this case, myName.age

This is can be compared to what happens with functions:

  • If we want to pass only the function itself, we only need to write it's name
  • If we want to pass some property of the function, we should write it afterwards, just like when we write parentheses

When we want to pass the function in it's entirety we SHOULD NOT use parentheses after it

Do you want to call the function or do you want to pass the function along? There is a huge difference between the two.

Most times dealing with callbacks, we just want to pass the function along, and not get it's result straight away.

Functions are Variables

To prove this statement we will use an example.

1º Example

Logging a function on the console

We create a function called myFunc, this function will be logged in the console.

We are not going to log this function in the conventional way

This function will be logged without ()

Using () after a function is what actually calls and executes the code inside the function

In this example we want to log the function itself, instead of calling it

                                            
function myFunc() {
    console.log("Hello World!")
}

console.log(myFunc)
Console: 
                                            
ƒ myFunc() {
    console.log("Hello World!")
}

The function was not called

What was logged was the actual definition of the myFunc function

To get a more useful representation from this function we can use console.dir

                                            
console.dir(myFunc)
Console: 
                                            
function helloWorld(value)
    arguments: null
    caller: null
    length: 1
    name: "helloWorld"
    prototype: Object { }
    <prototype>: function ()

Something that we can notice after using console.dir is that this function actually looks very similar to an Object

The similarities between functions and Objects do not end here

2º Example

Similarities between Functions and Objects

We create a function called myFunc, this function will be logged in the console.

We are not going to log this function in the conventional way

This function will be logged without ()

Using () after a function is what actually calls and executes the code inside the function

In this example we want to log the function itself, instead of calling it

                                            
function myFunc(value) {
    console.log("Hello World!")
}

const myObj = {
    name: "John",
    age: 32,
    married: true
}

console.dir(myFunc)
console.log(myObj)
Console: 
                                            
function myFunc(value)
    arguments: null
    caller: null
    length: 1
    name: "helloWorld"
    prototype: Object { … }
    <prototype>: function ()

Object
    age: 32
    married: true
    name: "John"
    <prototype>: Object { … }

After logging function and Object together, we can see that they are very similar

The reason behind this is that, in JavaScript, functions are actually Objects

Differences between functions and normal Objects

  • Functions are Objects that can be called by writing () in front of them
  • Functions are Objects that can have arguments passed to them

And What are Objects?

Let's remember an important factor:

Objects are Variables!

Objects are variables that you can pass around!

You can pass your objects anywhere you want:

  • You can pass them to a function;
  • You can call things that are inside of them;
  • You can change them;
  • You can do anything!

All of this is leading up to the fact that consequently functions are also variables!

Because of this, functions can also be passed anywhere we want to! Just like Objects, just like variables.
